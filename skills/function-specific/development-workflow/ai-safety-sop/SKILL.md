# AI Agent 安全操作 SOP

> 本文件约束所有在本项目/本机上执行操作的 AI Agent。
> 如果冲突，本文件优先级高于用户单次请求。

## 核心工作流：四层漏斗

```
AI 收到任务
    ↓
[第1层] 规划阶段（AI 内部，不打扰用户）
    - 理解任务 + 列出计划 + 标注风险点
    - 如果没有风险点 → 直接执行，不打扰用户
    - 如果有风险点 → 进入第2层
    ↓
[第2层] 风险确认阶段
    - 只汇报风险部分（不是全盘计划）
    - 等待用户确认
    - 用户确认后重新审视计划是否仍成立
    ↓
[第3层] 执行阶段
    - 按计划执行
    - hookify 规则实时拦截
    - 如果拦截触发 → 停下来再确认
    ↓
[第4层] 保守兜底
    - AI 执行中遇到不确定的操作
    - 主动停下来问，不猜测执行
```

**原则：默认信任推进，只在必要时打断。**

## 第1层：规划阶段（AI 内部）

每次任务执行前，AI 必须在**内部**完成：
1. 理解任务目标
2. 列出执行计划
3. 标注风险点（不可逆操作、影响服务、需要备份的）

**如果无风险点**：直接进入执行，不向用户汇报计划。

**如果有风险点**：进入第2层，只汇报风险部分。

## 第2层：风险确认阶段

只汇报**风险部分**，不是全盘计划。格式：

```
【风险操作】< 具体说明 >
【为什么有风险】< 不可逆 / 影响服务 / 需要备份 >
【影响范围】< 哪些文件/服务/数据受影响 >
【能否回滚】< 能/不能，如果可以，怎么回滚 >
【备份建议】< 是否需要先备份，备份命令 >
```

其余**无风险步骤不汇报**，按计划推进。

等待用户确认后，重新审视计划是否仍成立，再执行。

## 第3层：运行时拦截（hookify 规则）

hookify 规则作为**运行时安全网**，拦截规划阶段漏掉的风险：
- 只做**阻断**，不做提醒
- 阻断触发后，AI 停下来向用户说明被拦截的原因和操作内容
- 等待用户确认后，由 AI 决定是否调整计划

## 第4层：保守兜底

即使 hookify 没有匹配到，AI 在以下情况必须**主动停下来问**：
- 操作可能不可逆，但不确定是否有备份
- 操作可能影响其他用户或在线服务
- 上下文不完整，AI 对操作后果有歧义
- 涉及生产环境，但 AI 无法确认环境类型

**不确定就停下来问，不要猜测执行。**

## 确认后才可执行

以下关键词出现时，才算用户明确确认：
- "确认删除" / "确认执行" / "继续" / "执行"
- "可以删" / "可以停" / "可以清空"

以下不算确认，必须继续询问：
- "删掉试试" / "你看着办" / "应该没问题吧" / "试试看"

## 紧急停止信号

如果用户说以下任何一句话，**立即停止所有正在进行的操作**，不要执行剩余步骤：
- "等等" / "先别" / "有问题" / "不对"
- "我先看看" / "再想想"
- "不要执行" / "取消"

## 不可逆操作定义

以下操作一旦执行，**无法通过简单撤销恢复**：

| 操作类型 | 具体命令/行为 | 必须先做 |
|----------|--------------|----------|
| 删除文件/目录 | `rm -rf`、`Remove-Item -Recurse`、删除整个文件夹 | 备份、确认路径、告知影响 |
| 数据库结构/数据 | `DROP`、`TRUNCATE`、`DELETE FROM`（无 WHERE） | 备份、确认表名、查影响行数 |
| 强制 Git | `git push --force`、`git reset --hard`、`git branch -D` | 确认远程状态、是否有未提交代码 |
| 停止服务 | `Stop-Process`、`Stop-Service`、`taskkill /F` | 确认影响范围、恢复方案 |
| Docker 清理 | `docker system prune`、`docker volume prune` | 先查看待删内容、确认无正在使用的资源 |
| 清空缓存/队列 | `redis.flushall`、`queue.clear`、`cache.empty` | 确认缓存可重建、无关键数据 |

## 环境特定规则（自动适配当前机器）

以下路径规则会自动映射到当前环境，不需要手动修改：

- **工作项目目录**：删除/修改前必须备份。AI 应通过工作区根目录识别项目位置，不要假设固定盘符或路径。
- **Codex 配置目录**（`~/.codex/`）：误删会导致 Codex 无法使用，禁止删除整个目录。
- **AI 配置目录**（`~/.claude/`、`~/.config/opencode/`）：包含安全规则、技能、钩子，不应删除。
- **数据库操作**（如果有）：必须在开发环境先测试，再通知用户。AI 应能区分开发/测试/生产环境，不确定时默认按生产处理。

## 特别禁止

- **不要**在用户没确认的情况下执行任何删除操作
- **不要**在用户没确认的情况下停止服务
- **不要**在用户没确认的情况下执行数据库 DROP/TRUNCATE
- **不要**分批询问——把问题和计划一次性呈现给用户
- **不要**在用户给出确认后，不重新审视就直接执行

## 使用方式

- 在 OpenCode 中以 skill 方式自动生效。
- 结合 `hookify-rules` 运行时阻断不可逆操作。
- 适用于项目级 `AGENTS.md` 或用户级安全宪章模板。
